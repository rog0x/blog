How to Set Up MCP Servers in Claude Code: Complete 2026 Guide

March 21, 2026 · 8 min read · MCPClaude CodeTutorial

MCP (Model Context Protocol) is what turns Claude from a chatbot into a tool-using agent. With MCP servers, Claude can scrape websites, query databases, manage Docker containers, analyze GitHub repos, and much more — all through a standardized protocol.

This guide covers everything: what MCP is, how to install servers in Claude Code and Claude Desktop, configuration options, and the best servers to start with.

What Is MCP?

Model Context Protocol is an open standard created by Anthropic that lets AI models interact with external tools and data sources. Think of it like USB for AI — a universal connector that lets Claude plug into any tool that speaks the MCP protocol.

An MCP server is a small program that exposes tools to Claude. For example, an MCP server for web scraping might expose tools like web_extract, web_search, and web_monitor. Claude discovers these tools automatically and uses them when needed.

Prerequisites

  • Node.js 18+ — Required for running TypeScript MCP servers via npx
  • Claude Code or Claude Desktop — The client that connects to MCP servers
  • npm — Comes with Node.js, used to install MCP server packages

Check your Node version:

node --version  # Should be v18.0.0 or higher

Method 1: Claude Code CLI (Recommended)

The fastest way to add an MCP server. One command, done:

# Basic syntax
claude mcp add <name> <command> [args...]

# Real examples:
claude mcp add web-tools npx @rog0x/mcp-web-tools
claude mcp add seo-tools npx @rog0x/mcp-seo-tools
claude mcp add github-tools npx @rog0x/mcp-github-tools
claude mcp add json-tools npx @rog0x/mcp-json-tools
claude mcp add docker-tools npx @rog0x/mcp-docker-tools

After adding, restart Claude Code. The server starts automatically when Claude needs its tools.

Managing MCP Servers in Claude Code

# List all installed servers
claude mcp list

# Remove a server
claude mcp remove web-tools

# Check server status
claude mcp status

Method 2: Manual Configuration (settings.json)

For more control, edit the settings file directly. This is also required for Claude Desktop.

Claude Code

Edit ~/.claude/settings.json:

{
  "mcpServers": {
    "web-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-web-tools"]
    },
    "seo-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-seo-tools"]
    },
    "github-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-github-tools"]
    },
    "database-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-database-tools"]
    }
  }
}

Claude Desktop

Edit the config file at:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

Same format as above — add entries to mcpServers.

Method 3: Project-Level Configuration

Add MCP servers for a specific project only. Create .claude/settings.json in your project root:

{
  "mcpServers": {
    "testing-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-testing-tools"]
    },
    "git-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-git-tools"]
    }
  }
}

Project-level servers only activate when you open that project.

Passing Environment Variables

Some MCP servers need API keys or configuration:

{
  "mcpServers": {
    "github-tools": {
      "command": "npx",
      "args": ["@rog0x/mcp-github-tools"],
      "env": {
        "GITHUB_TOKEN": "ghp_your_token_here"
      }
    }
  }
}

Verifying Your Setup

After adding servers, verify they work:

  1. Restart Claude Code
  2. Ask Claude: "What MCP tools do you have available?"
  3. Claude will list all available tools from all connected servers
  4. Try a tool: "Use web_extract to get the title of https://example.com"

Troubleshooting

Server not appearing

  • Check claude mcp list — is the server listed?
  • Restart Claude Code after adding servers
  • Verify Node.js 18+ is installed

npx errors

  • Clear npx cache: npx --yes clear-npx-cache
  • Try installing globally: npm install -g @rog0x/mcp-web-tools
  • Check npm registry access: npm ping

Permission errors

  • On macOS/Linux: avoid sudo for npx. Fix npm permissions instead.
  • On Windows: run terminal as administrator if needed
