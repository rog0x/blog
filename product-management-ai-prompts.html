AI Prompts for Product Managers

March 20, 2026 · 7 min read · ProductAIManagement

Product managers write more docs than engineers. PRDs, roadmaps, user stories, release notes, stakeholder updates, competitive analyses — the list never ends. AI handles 80% of the first draft so you can focus on the 20% that requires judgment.

The PRD Generator

Write a PRD for: [FEATURE]

Product: [PRODUCT]
Target user: [USER]
Business goal: [GOAL]

Sections:
1. Overview (what and why — 2 paragraphs)
2. User stories (5-8, "As a [user], I want...")
3. Requirements (functional + non-functional)
4. Success metrics (3-5 measurable outcomes)
5. Out of scope (what we're NOT building)
6. Open questions (decisions still needed)
7. Timeline (phases with milestones)

Keep it under 3 pages. No fluff.

The "out of scope" section prevents scope creep before it starts. The "open questions" section surfaces decisions you haven't made yet — better to find gaps now than mid-sprint.

Prioritization with RICE

Prioritize these features using RICE:

Features: [list features]
Context: [product stage, team size, goals]

For each feature, estimate:
- Reach: how many users affected per quarter
- Impact: 0.25 (minimal) to 3 (massive)
- Confidence: 50% to 100%
- Effort: person-weeks

Calculate RICE score = (Reach × Impact × Confidence) / Effort

Rank by score. Flag any with low confidence for research.

User Interview Script

Create a user interview script for: [TOPIC]

Target user: [USER TYPE]
Duration: 30 minutes

Rules:
- Ask about past behavior, not future intentions
- "Would you use X?" → useless
- "How did you solve X last time?" → gold
- Don't lead the witness
- Follow up with "why?" at least twice

Include: intro (2 min), problem exploration (10 min),
current solutions (10 min), reaction to concept (5 min),
wrap-up (3 min)

Metrics Dashboard Design

Design a product metrics dashboard for [PRODUCT]:

Key metrics (5 max):
1. [Metric 1] — what it measures, how to calculate
2. [Metric 2]
...

For each metric:
- Current value and target
- Trend direction (improving/declining/flat)
- Alert threshold (when to worry)
- Visualization type (line, bar, number, gauge)

Dashboard layout: 4 panels on one screen.
Update frequency: daily / weekly / real-time

Go-to-Market Plan

Create a GTM plan for [FEATURE/PRODUCT]:

Launch date: [DATE]
Audience: [WHO]
Key message: [ONE SENTENCE]

Plan:
1. Pre-launch (T-14 to T-1): what to prepare
2. Launch day: channels, messaging, timing
3. Week 1: monitor, respond, adjust
4. Month 1: measure, iterate, expand

Channels: email, social, blog, product, partners
Success metric: [WHAT DEFINES SUCCESS]
