How to Run a Security Audit with AI Prompts

March 20, 2026 · 7 min read · SecurityAIOWASP

Most applications ship with security vulnerabilities. Not because developers don't care, but because security reviews take time nobody has. AI changes that equation — a thorough security audit that used to take days now takes hours.

Here are the prompts that find real vulnerabilities.

The OWASP Top 10 Audit

Audit this code for OWASP Top 10 vulnerabilities:

[paste code or describe the application]

Check for:
1. Injection (SQL, NoSQL, OS command, LDAP)
2. Broken Authentication
3. Sensitive Data Exposure
4. XML External Entities (XXE)
5. Broken Access Control
6. Security Misconfiguration
7. Cross-Site Scripting (XSS)
8. Insecure Deserialization
9. Using Components with Known Vulnerabilities
10. Insufficient Logging & Monitoring

For each finding:
- CWE identifier
- Risk level (Critical/High/Medium/Low)
- Proof of concept
- Remediation code

The structured checklist ensures AI doesn't skip categories. The "proof of concept" requirement forces concrete findings instead of vague warnings.

SQL Injection Scanner

Find all SQL injection vectors in this code:

[paste code]

For each vector:
1. The vulnerable line/function
2. What user input reaches the query
3. Example malicious input
4. The fix (parameterized query)

Also check for:
- ORM misuse (raw queries with string interpolation)
- Dynamic table/column names from user input
- LIKE clauses with unescaped wildcards

Authentication Audit

Audit the authentication system:

[describe or paste auth code]

Check:
1. Password hashing (bcrypt? salt? cost factor?)
2. Session management (token type, expiry, storage)
3. Brute force protection (rate limiting, lockout)
4. Password reset flow (token expiry, one-time use)
5. JWT implementation (algorithm, expiry, refresh flow)
6. OAuth implementation (state parameter, redirect validation)
7. 2FA implementation (backup codes, rate limiting)

For each issue: severity, exploit scenario, fix.

Dependency Audit

Audit dependencies in this package.json:

[paste package.json]

Check:
1. Known CVEs in current versions
2. Packages that haven't been updated in 2+ years
3. Packages with very few maintainers (bus factor)
4. Packages that are deprecated
5. Unnecessary dependencies (do we really need lodash?)
6. Dev dependencies accidentally in production

For each issue: severity, what to upgrade to, breaking changes to watch for.

Infrastructure Security

Audit this server/deployment configuration:

[paste nginx config, docker-compose, k8s manifest, etc.]

Check:
1. TLS configuration (version, ciphers, HSTS)
2. Security headers (CSP, X-Frame, X-Content-Type)
3. CORS policy (too permissive?)
4. Rate limiting (present? configured?)
5. Secrets in config files (API keys, passwords)
6. Default credentials
7. Debug mode in production
8. Unnecessary open ports
9. Non-root user for services
10. Log sanitization (no sensitive data in logs)

The Pattern

All effective security prompts share three traits:

  1. Specific checklist. Don't say "check for security issues." List exactly what to check.
  2. Proof required. "Show the vulnerable line and a malicious input" prevents false positives.
  3. Fix included. Every finding must come with remediation code, not just a warning.
